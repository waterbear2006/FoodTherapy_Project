# 基于图深度优先搜索的食疗图谱推荐算法 (DFS Recipe Graph Recommendation) 实施完成与算法原理解析

我们在“每日养生推荐引擎 (`daily_report_engine.py`)”中结合知识图谱实现了复杂的自动食疗方剂生成闭环。

针对“**当收集到对应体质与节气后，如何把合适的食材拼凑并推导出对应的有效菜谱**”这一核心诉求，我们自主研发了一套**环境自适应的多因子特征映射与图谱深度检索算法**，以下为原理解析。

## 核心算法架构：多特征评估与受限深度优先图检索

中医方块学究的本质是用“药材（对应本需求内食材）”组合达成某种功能主治“方剂（对应本需求菜谱）”。
我们完全抛弃了死板的“数据库联表一对多硬编码”匹配，转向了更具通用智能与网状扩展性的**图论检索算法策略**。

### 1. 食材-菜谱二分图知识网络构建 (Bipartite Graph Construction)
算法初始化（`_build_graph`）时，会将所有内存加载好的原材料提取完毕。
我们对每一种“菜谱（Recipe）”打散，拉出它所需要的材料清单，与庞大的“食材分类表（Ingredient）”作遍历比对。算法将建立一个巨大的无向连通词典：
```python
# 核心构建片段
self.graph.add_edge(ingredient_name, recipe_name)
```
从此，食材与配方具备了**拓扑相邻性 (Topological Adjacency)**。

### 2. 强体质与动态季节环境因子的线性特征惩戒与打分 (Multi-Factor Weight Ranking)
有了网状关系图后，需要找到探索的“起点（原发最优食材）”。
算法提取当前用户的主体质（如“阴虚质”）获取其目标主攻方向数组（如`["滋阴", "清热"]`），并叠加当前日期的节气季候增强子（如秋季增加"润燥"需求）。这部分叠加后与环境气候因子（详见“动态气象权重因子算法”）整合，针对所有的游离食材库展开了一场特征计分大赛：
> `基础分 = 功效标签重叠数(tag_hits) * 2 + 强体质适用性权重补充 + 天气环境奖励系数(weather_bonus)`

算法通过得分降序排序，提取出真正针对当下状态、季候以及气候的“绝佳 Top-K 基底食材集合（`available_ingredients`）”。

### 3. 图节点受限深度优先搜索策略 (Bounded DFS Graph Traversal)
获取了若干个最合适当前发热或虚寒状态的基础核心食材后，算法以此节点输入，下潜进入我们预先搭建好的那张“知识连通图”中寻找拼图解法：

在此阶段引擎使用经典的 **深度优先搜索 (Depth First Search)**。由于菜谱之间亦可能存在共同食材重叠的爆炸式联接，我们严格执行了以下剪枝法则：
- **深度封锁 (`max_depth = 2`)**：杜绝进入极其复杂的套娃迭代中。算法只需检索食材旁邻的一度菜谱，或包含该一度菜谱的其他材料作为验证即可。
- **体质负面排除过滤**：遍历触碰到任何邻居菜谱节点时，在计入前，前置探测其 `suitable_list` 适用偏好，禁忌直接掐除分支。
- **动态路径成分回溯收集 (`path_ingredients | {next_ingredient}`)**：记录 DFS 的历史探寻路径以收集拼凑情况。

### 4. 匹配率量化与局部优解输出 (Match Rate Evaluation & Sorting)
在遍历搜集完毕该起始原发食材关联到的“可能匹配群谱”后，算法并没有盲抛结果。
针对触碰的所有配方引入了数学评价公式：**匹配率 (Match Rate) = `已满足搜集食材总图深度数 / 该单体菜谱需求的材料总量`**。

所有的配方池将会基于这套 **匹配率** 与其 **组方成分丰饶度** 的联合指标，进行二次降序（Sort Descending）。这保证了，最终能浮现到用户 `[推荐的菜谱]` 集合首页的结果是：
不仅仅其中一两味主药精确对上了你的体质和时令，它整套方子的组方材料覆盖面积在全局范围内亦达到了最贴切的最优解，实现了高水准的“据症投食”。
