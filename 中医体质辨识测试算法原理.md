# 中医体质辨识测试算法 (TCM Constitution Identification Algorithm) 实施完成与算法原理解析

我们已经成功将“中医九种体质判定系统”集成入到了我们后端的体质测试引擎 (`quiz_engine.py`) 中，并扩展了 API 接口以支持快速的在线体质评估。

在这里，针对此项目中的核心功能：“**如何根据用户的答卷收集数据，并转化为标准化中医体质判定结果**”，进行详尽的算法原理解析。

## 核心算法架构：标准化分数转换与规则驱动判定 (Standardized Score Conversion & Rule-Based Classification)

在处理用户的测量问卷数据时，算法核心基于国家中医药管理局发布的 **2009版《中医体质分类与判定》标准**。这是一个典型的基于统计与专家规则组合的线性决策模型。

整个闭环评估算法可以解构为以下 3 个核心阶段：

### 1. 原始分归置与汇总 (Raw Score Aggregation)
用户答卷本质是一个包含了若干问题与其权重的离散化向量组。
在引擎的算力分发逻辑中，首先会对 `answers` 数组进行遍历聚合，将打散的答案按其所属的“体质类别目录 (`category`)” 收拢：
```python
category_stats = {c: {"sum": 0, "count": 0} for c in self.constitutions}
```
此时能得出每个类别下该用户的“原始分总数”以及该体质考察的“有效题目总数”。

### 2. 标准化测试转化分数计算 (Standardized Score Conversion Matrix)
这是极其重要的一步。由于“平和质”和“气虚质”等不同体质在题库里的考察题目数量可能并不相同（有的题多有的题少），直接使用原始分比对将会出现严重的尺度失衡（Scale Imbalance）。
算法在此引入了“极差转化方程”，将任意题目数的原始分强制映射贴合到 `0~100` 的标准区间中：
> **转化分 = [(原始分 - 题目数) / (题目数 * 4)] * 100**

这一转换直接统一了9个维度的评价标尺（`[0, 100]`），生成了当前患者的**正交化体质特征向量 (Score Vector)**，允许不同体质的分数进行直接横向比较。

### 3. 多维度体质类型判定逻辑 (Multi-dimensional Rule Decision Logic)
基于转换后的体征向量集，算法引擎调用《中医体质判定标准》的最高专家优先级规则，进行最终定性判定。这部分逻辑包含两类处理路径：

- **A. 偏颇体质倾向辨识 (Imbalanced Constitution Detection)**
  剥离“平和质”，只扫描其余8种亚健康/病前偏颇体质。
  - **确诊区间**：一旦某项体质得分 `>= 40`，算法直接推断确诊具备该体质，写入 `results`。
  - **倾向区间**：倘若所有得分均低于40阈值，算法降级检测 `30-39` 范围。如果有命中，说明病情潜在蕴含，判定诊断为“具有该体质倾向”。

- **B. 平和质竞争互斥判定 (Pinghe-zhi Exclusive Evaluation)**
  平和质代表机体健康状态，其触发条件在算法判定树中最为严苛：
  - 前置条件：必须在步骤A中没有任何确定的偏颇或偏颇倾向（8种体质转化分均严格 `< 30`）。
  - 若最高偏颇得分 `< 30`：此时比较“平和质分数”是否 `>= 60`，是则诊断为完美均态“平和质”。
  - **次级底线兜底**：如果平和质依旧不达要求甚至得分十分低靡。该函数进行最终妥协，强制取那8项哪怕不足30分的偏颇体质中的最高分那一项强行作为主导，或者取两者相对高值得出结论。

## 优化与工程落地
1. **可回溯特征向量保留** (`constitution_vector`)：算法不但输出直接结果，还会向前端传递各体质维度的准确分值对，以便前端渲染类似“雷达图”之类的多维直观反馈。
2. **复合型体质处理支持** (`is_combination`)：如果用户的 `[高分集合]` 中存在复数项超标特征，算法默认兼容“兼夹体质”逻辑，使得如“主体质为气虚质，并带有痰湿倾向”的复杂中医确诊成为可能并在 `description` 中优雅响应给终端用户。
